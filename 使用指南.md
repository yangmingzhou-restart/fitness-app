## 2. 第一步：启动后端服务

后端的作用：接收手机发来的照片 → 调用阿里云 Qwen AI 分析 → 返回结果

### 2.1 确认 API Key 已配置

检查 `backend/.env` 文件，确保已填入你的阿里云百炼 API Key：

```
DASHSCOPE_API_KEY=sk-你的真实Key
```

> 注意：你的 Key 已存在于文件中，无需再次配置。

### 2.2 启动后端

**方法一：双击启动脚本（最简单）**

双击 `backend/start.bat` 文件。

**方法二：命令行启动**

```bash
cd backend
D:\Anaconda3\python.exe -u main.py
```

### 2.3 验证后端是否启动成功

看到以下输出即表示成功：

```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Started server process [xxxxx]
2026-05-08 xx:xx:xx,xxx - main - INFO - 正在初始化数据库...
2026-05-08 xx:xx:xx,xxx - main - INFO - 数据库初始化完成
```

### 2.4 保持后端运行

**这个命令行窗口不要关！** 后端必须一直运行。

## 3. 第二步：安装 ngrok（手机访问用）

> 为什么需要 ngrok？
> 
> 你的后端跑在电脑的 localhost:8000，手机没法直接访问。ngrok 可以把 localhost 映射为一个公网地址（https://xxx.ngrok.io），手机通过 4G/5G/WiFi 都能访问。

### 3.1 下载 ngrok

1. 打开 https://ngrok.com/download
2. 下载 Windows 版本（zip 文件）
3. 解压到一个目录，比如 `C:\ngrok\`

### 3.2 注册 ngrok 账号（免费）

1. 打开 https://dashboard.ngrok.com/signup
2. 可以用 Google/GitHub 账号快速登录
3. 登录后进入 https://dashboard.ngrok.com/get-started/your-authtoken
4. 复制你的 Authtoken（一串 `xxxxx_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx` 格式的文本）

### 3.3 配置 ngrok Authtoken

```bash
# 在命令行中运行（替换为你的 authtoken）
ngrok config add-authtoken 你的_authtoken_字符串
```

## 4. 第三步：启动 ngrok 隧道

**前提：后端已经在运行！**（第1步的窗口不要关）

打开 **一个新的命令行窗口**，运行：

```bash
ngrok http 8000
```

看到以下输出即成功：

```
Forwarding  https://xxxx.ngrok.io -> http://localhost:8000
```

**关键：复制这个 https://xxxx.ngrok.io 地址，后面要用！**

> 注意：免费版 ngrok 每次启动地址都会变，所以每次使用都要重新配置前端。

**保持 ngrok 窗口不关！**

---

## 5. 第四步：配置前端 API 地址

### 5.1 找到前端 API 配置文件

文件位置：`frontend/src/config/api.ts`

### 5.2 修改 API 地址

```typescript
// 把 DEV_API_URL 改为你的 ngrok 地址
// 例如：
export const API_BASE_URL = 'https://xxxx.ngrok.io'
```

或者更方便的方法——**不用改代码，用环境变量**：

```bash
# 在启动前端时，加上 ngrok 地址
cd frontend
set EXPO_PUBLIC_API_URL=https://xxxx.ngrok.io && npx expo start
```

### 5.3 重要：手机上也要能访问 ngrok 地址

在手机浏览器打开 `https://xxxx.ngrok.io/health`，看到 `{"status":"ok"}` 说明网络通了。

## 6. 第五步：启动前端 App

**前提：后端已在运行、ngrok 已启动**

打开 **又一个新的命令行窗口**，运行：

```bash
cd frontend
npx expo start
```

看到类似这样的输出：

```
› Metro waiting on exp://xxxxx
› Scan the QR code above with Expo Go (Android) or the Camera app (iOS)
```

### 扫码连接

- **iPhone**：打开系统相机，扫二维码 → 点击通知打开 Expo Go
- **Android**：打开 Expo Go App → 扫二维码

等待几秒钟，App 就会加载到你的手机上。

## 7. 第六步：手机连接测试

### 7.1 确认连接

App 加载到手机上后，你会看到：
- 顶部：**"食物热量识别"** 标题 + **"EN"** 语言切换按钮

## 8. App 使用流程

### 完整使用流程

```
1. 手机打开 App
2. 对准盘子里的食物
          ↓
3. 拍第1张（俯拍，显示食物全貌）
          ↓
4. 可选：点"再加一张"，换个角度拍
          ↓
5. 可选：拍第3张、第4张（侧拍，显示厚度/容器深度）
          ↓
6. 点绿色按钮 "分析食物（X张）"
          ↓
7. 等待3-5秒
          ↓
8. 看到结果：
   ┌─────────────────────┐
   │  食物：清炒西兰花     │
   │  热量：350 kcal/kg   │
   │  估计份量：250g       │
   │  估计热量：87.5 kcal  │
   │  置信度：95%          │
   │  总热量：87.5 kcal    │
   │                      │
   │  [重新拍照]           │
   └─────────────────────┘
          ↓
9. 自动保存到历史记录
          ↓
10. 点底部"历史记录"可查看
```

### 拍照技巧（提高准确率）

| 技巧 | 说明 |
|------|------|
| **光线充足** | 自然光最佳，避免阴影遮挡食物 |
| **俯拍一张** | 从上往下拍，展示食物全貌和摆盘 |
| **侧拍一张** | 从45度角拍，展示食物厚度和容器深度 |
| **不同角度** | 2-3张不同角度最佳，4张封顶 |
| **背景简洁** | 避免杂乱背景干扰识别 |
| **对焦清晰** | 确保食物在焦点上，不模糊 |

### 语言切换

点击顶部 **"EN"** 按钮，界面立即切换为英文；
再点一次 **"中文"** 切换回中文。

## 10. 常见问题排查

### Q: 后端启动报错 "ModuleNotFoundError"

```bash
# 重新安装依赖
conda run -n base pip install fastapi uvicorn python-multipart aiofiles openai python-dotenv Pillow
```

### Q: 手机扫码后 App 一直加载中

1. 检查手机和电脑是否**在同一网络**（如果用局域网方式）
2. 如果手机用 4G/5G：确保 ngrok 已正确启动
3. 在手机浏览器打开 `https://你的ngrok地址/health` 测试连通性

### Q: 分析结果一直不出来

1. 检查 `backend/.env` 中的 **DASHSCOPE_API_KEY** 是否正确
2. 检查后端命令行窗口是否有错误日志
3. 确认网络可以访问阿里云

### Q: 想换电脑/换网络重新用

每次重新使用的步骤：

```bash
# 1. 启动后端
cd 健身app/backend
D:\Anaconda3\python.exe -u main.py

# 2. 新开窗口，启动 ngrok
ngrok http 8000

# 3. 复制 ngrok 地址
# 4. 新开窗口，启动前端（带上 ngrok 地址）
set EXPO_PUBLIC_API_URL=https://新的ngrok地址 && npx expo start

# 5. 手机扫码使用
```

### Q: 想迁移到云端服务器

1. 把 `backend/` 整个目录上传到云服务器（阿里云ECS等）
2. 安装依赖
3. 把 `.env` 的配置改一下（数据库可以换成 PostgreSQL）
4. 用 `uvicorn main:app --host 0.0.0.0 --port 8000` 启动
5. 前端只需要改 `frontend/src/config/api.ts` 中的 `API_BASE_URL` 为云服务器地址

### Q: 关闭电脑后还能用吗？

不能。后端和 ngrok 都运行在你的电脑上，关机后服务就停了。
如果要 24/7 使用，需要将后端部署到云端服务器（见上一条）。
